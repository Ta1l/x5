# Настройка Firebase для сохранения заявок

## Шаг 1: Создание проекта Firebase

1. Перейдите на [https://console.firebase.google.com](https://console.firebase.google.com)
2. Нажмите **"Добавить проект"**
3. Введите имя проекта (например, "Courier-App")
4. Нажмите **"Создать проект"**
5. Дождитесь завершения инициализации

## Шаг 2: Настройка Firestore Database

1. В левом меню выберите **"Firestore Database"**
2. Нажмите **"Создать базу данных"**
3. Выберите режим **"Начать в режиме тестирования"**
4. Выберите регион (**europe-west1** для Европы рекомендуется)
5. Нажмите **"Создать"**

## Шаг 3: Получение конфигурации Firebase

1. Нажмите на значок шестеренки в левом меню → **"Настройки проекта"**
2. Перейдите на вкладку **"Ваши приложения"**
3. Нажмите **"Добавить приложение"** → выберите **"Веб"** (</> значок)
4. Введите имя приложения
5. Скопируйте код конфигурации

Конфигурация будет выглядеть так:
```javascript
const firebaseConfig = {
  apiKey: "ВАШ_API_KEY",
  authDomain: "ВАШ_ПРОЕКТ.firebaseapp.com",
  projectId: "ВАШ_PROJECT_ID",
  storageBucket: "ВАШ_ПРОЕКТ.appspot.com",
  messagingSenderId: "ВАШ_MESSAGING_SENDER_ID",
  appId: "ВАШ_APP_ID"
};
```

## Шаг 4: Обновление конфигурации в HTML

В файле `index.html` найдите блок кода с конфигурацией Firebase (примерно строка 30):

```javascript
const firebaseConfig = {
  apiKey: "YOUR_API_KEY",
  authDomain: "YOUR_PROJECT.firebaseapp.com",
  projectId: "YOUR_PROJECT_ID",
  storageBucket: "YOUR_PROJECT.appspot.com",
  messagingSenderId: "YOUR_MESSAGING_SENDER_ID",
  appId: "YOUR_APP_ID"
};
```

Замените значения на ваши реальные значения из консоли Firebase.

## Шаг 5: Проверка безопасности (важно!)

**⚠️ ВНИМАНИЕ:** После разработки и тестирования измените правила безопасности!

1. В Firestore перейдите на вкладку **"Правила"**
2. Модифицируйте правила для ограничения доступа (сейчас в режиме тестирования они открыты для всех):

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /courier_applications/{document=**} {
      allow create: if true;
      allow read, update, delete: if false;
    }
  }
}
```

Это позволит создавать новые заявки, но запретит чтение и удаление.

## Шаг 6: Проверка сохранения данных

1. Заполните форму на сайте и отправьте заявку
2. Вернитесь в Firestore Console
3. Перейдите в коллекцию **`courier_applications`**
4. Проверьте, что заявки появляются там в реальном времени

## Как просмотреть сохраненные заявки

Все заявки хранятся в коллекции `courier_applications` с полями:
- `name` - Имя курьера
- `phone` - Номер телефона
- `citizenship` - Гражданство
- `messenger` - Предпочитаемый мессенджер
- `timestamp` - ISO временная метка
- `submittedAt` - Дата и время в формате РФ

## Переменные среды для production (опционально)

Для дополнительной безопасности в production можно использовать переменные окружения. Но для этого нужен build process. У статического сайта такой возможности нет - поэтому убедитесь, что правила Firestore правильно настроены на сохранение только данных заявок.

## Troubleshooting

**Проблема:** "Firebase is not defined"
- **Решение:** Проверьте, что конфигурация добавлена правильно и нет ошибок в значениях

**Проблема:** Заявки не сохраняются
- **Решение:** Проверьте правила Firestore - убедитесь, что режим тестирования включен или правила позволяют создавать документы

**Проблема:** CORS ошибка
- **Решение:** Это нормально для локальной разработки. На production сервере это не должно быть проблемой

## Резервное копирование данных

Данные также сохраняются в `localStorage` браузера в случае, если Firebase недоступна. Вы можете проверить их в консоли браузера (F12 → Application → Local Storage).
