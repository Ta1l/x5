# ⚡ БЫСТРАЯ ИНСТРУКЦИЯ: ЧТО НАДО СДЕЛАТЬ ПРЯМО СЕЙЧАС

**Это краткая инструкция что нужно сделать ДО запуска развертывания на сервере**

---

## 🔴 КРИТИЧЕСКИЕ ПРОВЕРКИ (ДЕЛАЙТЕ СЕЙЧАС!)

### 1️⃣ DNS НАСТРОЙКА - САМОЕ ВАЖНОЕ!

**На вашей локальной машине проверьте:**

```bash
nslookup slotworker.ru
```

**Ожидаемый результат:**
```
Адрес: 62.217.182.74
```

**ЕСЛИ ПОКАЗАЛО ДРУГОЙ IP ИЛИ "Non-existent domain":**

1. Перейти в панель регистратора домена (Reg.ru, Yandex, и т.д.)
2. Найти управление DNS / Name servers
3. Добавить две A записи:
   ```
   Тип: A
   Имя: slotworker.ru
   Значение: 62.217.182.74
   TTL: 3600
   
   Тип: A
   Имя: www.slotworker.ru
   Значение: 62.217.182.74
   TTL: 3600
   ```
4. **ДОЖДИТЕСЬ 5-24 часа** пока DNS распространится
5. Проверьте еще раз: `nslookup slotworker.ru`

❗ **БЕЗ ЭТОГО ШАГА САЙТ НЕ БУДЕТ ДОСТУПЕН ПО ДОМЕНУ!**

---

### 2️⃣ ПРОВЕРКА SSH ДОСТУПА

**На вашей локальной машине:**

```bash
ssh root@62.217.182.74
```

**Введите пароль:** `*9w1Z*!R7WxH`

**Ожидаемый результат:**
```bash
root@server:~#
```

**Если не подключается:**

- ✅ Проверьте что вводите пароль правильно
- ✅ Проверьте IP адрес (62.217.182.74)
- ✅ Убедитесь что сервер включен
- ✅ Проверьте ISP firewall (может блокировать порт 22)

Если используется прокси:
```bash
ssh -o ProxyCommand='nc -X 5 -x 6NeZMV:iSxcP9mEj0@46.8.17.103:5501 %h %p' root@62.217.182.74
```

---

### 3️⃣ ПРОВЕРКА ПОРТОВ

**На сервере выполните:**

```bash
# Проверить что портов доступны
netstat -tuln | grep -E ":80|:443"
```

**Ожидаемый результат:**
- Порт 80 слушается
- Порт 443 слушается

**Если не видите портов:**
- Может быть используются другим сервисом
- Придется их освободить или использовать другие

---

### 4️⃣ ПРОВЕРКА ИНТЕРНЕТА НА СЕРВЕРЕ

**На сервере:**

```bash
ping 8.8.8.8 -c 5
curl -I https://www.google.com
```

**Ожидаемый результат:**
- ping получает ответы
- curl показывает 200 OK

**Если не работает интернет:**
- Помощь провайдера хостера

## ❌ ЕСЛИ ЧТО-ТО ПОШЛО НЕ ТАК

**1. Проверить логи приложения:**
```bash
sudo journalctl -u courier-app -n 50
```

**2. Проверить Nginx:**
```bash
sudo systemctl status nginx
sudo nginx -t
sudo tail -30 /var/log/nginx/error.log
```

**3. Проверить что прослушивается порты:**
```bash
sudo ss -tlnp | grep -E ":80|:443|:3000"
```

**4. Полная диагностика:**
```bash
bash /var/www/x5/server-diagnostic.sh
```

**5. Если все плохо - перезагрузить всё:**
```bash
sudo systemctl restart courier-app
sudo systemctl restart nginx
sudo systemctl status courier-app
sudo systemctl status nginx
```
